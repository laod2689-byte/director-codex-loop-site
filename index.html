Multi-Agent Coding Workflow

Director-Codex Loop

一个把“需求理解、任务拆解、代码执行、测试验证、结果审查”串成同一闭环的双 Agent 工程系统。 它解决的不是单次生成代码的问题，而是如何让 AI 在真实仓库里完成可验证、可迭代、可追溯的交付。

Claude 负责计划与验收 Codex 负责实现与修复 测试与 git diff 自动回流 不通过就进入下一轮修正

What This Project Changes

把 AI 从“回答工具”升级为“能交付结果的工程代理”

项目的价值不在于让模型写一段代码，而在于把计划、执行、验证、审批组织成一套稳定流程。 这意味着复杂任务不再依赖人工手工串联，而是通过结构化工件和审查回路持续推进。

Role Separation

Director 不写代码，只定义目标、约束、风险和验收标准；Worker 不做抽象管理，只按计划落地。

Evidence-Driven Approval

审批依据不是主观感觉，而是代码变更、测试结果、执行报告和差异证据。

Repeatable Delivery

每轮运行都会保留计划、审查和最终结果文件，适合复盘、演示和持续扩展。

项目解决的核心痛点

在真实开发里，需求理解、验收定义、代码修改、测试验证、结果审查通常是分散的，人工需要在多个工具之间来回切换。 这会导致三个常见问题：第一，需求拆得不够细，模型容易“看起来完成了，实际上没有交付”；第二，测试与审查脱节，缺少统一裁决标准；第三，多轮修复时上下文丢失，迭代效率迅速下降。

这套系统的目标

  • 让任务在进入执行前就拥有明确、可验证的验收标准。
  • 让代码修改、测试结果和审查意见形成自动闭环，而不是散落在聊天记录里。
  • 让复杂任务具备长链推理能力，不再停留在单轮问答式辅助。
  • 让整个交付过程留痕，便于展示、复盘和后续扩展为更复杂的 Agent 网络。
核心逻辑流
Step 1 任务输入

用户输入自然语言目标、仓库路径和测试命令。

Step 2 Director 规划

Claude 提炼约束、显式列出假设、风险和验收标准。

Step 3 Worker 执行

Codex 在目标仓库中直接完成代码修改或文件生成。

Step 4 自动采证

控制器收集测试结果、执行日志和 git diff。

Step 5 审查裁决

Claude 根据结构化证据决定批准或退回修改。

Step 6 闭环迭代

未通过则进入下一轮，直到满足交付标准。

长链推理

这个系统不是“给提示词 -> 出答案”的一次性流程，而是先理解问题、再定义成功标准、再执行和验证，天然跨越多个状态与阶段。

多 Agent 协作

Director 与 Worker 职责硬分离，减少模型自评偏差，也让流程更容易扩展到更多专职 Agent。

结构化留痕

每轮都会生成标准化 JSON 工件，适合自动化系统、可视化面板和后续审计使用。

Smoke Repo 自动修复闭环

系统在 smoke 仓库中处理了一个针对 greet 函数的任务。流程先由 Director 生成计划， 再由 Worker 检查代码和测试，随后执行 python -m pytest -q 完成验证，最终在第 1 轮通过审批。

  • 结果状态为 approved
  • 测试命令执行通过，返回码为 0
  • 变更范围被限制在目标任务内

为什么这个项目适合对外展示

它不是抽象概念页，而是一个已经跑过真实任务、保留了真实运行证据的 AI 工程系统。对外展示时，最有说服力的不是“我用了大模型”， 而是“我让模型在明确约束下完成了可审查的交付，并且整个过程可回放、可核验、可迭代”。这正是 Director-Codex Loop 的核心价值。
